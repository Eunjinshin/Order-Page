## Phase 1 — 백엔드 기반 세팅 (Spring Boot 3.x)

### 1-1. 프로젝트 생성
**위치**: `backend/` (worktree 루트 하위)

**build.gradle 핵심 의존성**:
```
spring-boot-starter-web
spring-boot-starter-data-jpa
spring-boot-starter-security
spring-boot-starter-data-redis
spring-boot-starter-validation
spring-boot-starter-mail
flyway-core + flyway-mysql
mysql-connector-j
lombok
springdoc-openapi-starter-webmvc-ui (Swagger)
jjwt (JWT 처리)
```

**Java 버전**: 17
**Spring Boot 버전**: 3.3.x

### 1-2. 패키지 구조
```
backend/src/main/java/com/odersite/
├── config/          # Security, Redis, Swagger, Flyway 설정
├── domain/
│   ├── auth/        # 인증 (entity, repository, service, controller)
│   ├── member/      # 회원 (entity, repository, service, controller)
│   ├── product/     # 상품
│   ├── order/       # 주문
│   ├── payment/     # 결제
│   ├── delivery/    # 배송
│   ├── review/      # 리뷰
│   └── coupon/      # 쿠폰
├── global/
│   ├── exception/   # 전역 예외 처리
│   ├── response/    # 공통 응답 포맷
│   └── util/        # JWT 유틸
└── OdersiteApplication.java
```

### 1-3. 설정 파일 (application.yml)
```yaml
# MySQL 8.0 연결
spring.datasource:
  url: jdbc:mysql://localhost:3306/odersite?serverTimezone=Asia/Seoul
  username: odersite
  password: (환경변수)
  driver-class-name: com.mysql.cj.jdbc.Driver

# JPA
spring.jpa:
  hibernate.ddl-auto: validate  # Flyway가 스키마 관리
  show-sql: true

# Redis 7.x
spring.data.redis:
  host: localhost
  port: 6379

# Flyway
spring.flyway:
  enabled: true
  locations: classpath:db/migration

# JWT (환경변수)
jwt:
  secret: ${JWT_SECRET}
  access-expiration: 3600000     # 1시간
  refresh-expiration: 2592000000 # 30일
```

### 1-4. Docker Compose (로컬 개발 환경)
**파일**: `docker-compose.yml` (worktree 루트)
```yaml
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_DATABASE: odersite
      MYSQL_USER: odersite
      MYSQL_PASSWORD: odersite123
    ports: ["3306:3306"]
    volumes: [mysql_data:/var/lib/mysql]

  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]
```

### 1-5. Flyway 마이그레이션
**파일**: `backend/src/main/resources/db/migration/V1__init_schema.sql`

ERD v1.0 기준 23개 테이블 전체 DDL:
- 순서: MEMBER_USER → AUTH_LOGIN → MEMBER_PROFILE → MEMBER_ADDRESS
         → SHOP → CATEGORY → PRODUCT → PRODUCT_OPTION → PRODUCT_IMAGE → STOCK_HISTORY
         → COUPON → USER_COUPON
         → CART → ORDERS → ORDER_ITEM → RETURN_REQUEST
         → PAYMENT → REFUND → DELIVERY
         → REVIEW → REVIEW_IMAGE → REVIEW_HELPFUL → REVIEW_REPLY
- 인덱스 포함 (P1/P2/P3 우선순위 전체)
- ENUM 타입, FK 제약, Unique 인덱스 포함

**미확인 사항**: MySQL ENUM vs VARCHAR + CHECK CONSTRAINT 중 어떤 방식으로 구현할지 — ERD 상 ENUM 표기를 따라 MySQL ENUM으로 진행.

### 1-6. 기본 Entity (도메인별 1개씩 골격만)

- `AuthLogin.java`
- JPA Auditing 공통 BaseEntity (`createdAt` 자동화)

## 검증 방법

### 백엔드 검증
1. `docker-compose up -d` → MySQL + Redis 실행 확인
2. `./gradlew bootRun` → 서버 기동 확인 (포트 8080)
3. Flyway 마이그레이션 성공 로그 확인 (`Successfully applied 1 migration`)
4. `http://localhost:8080/swagger-ui.html` 접속 확인

### 프론트엔드 검증
1. `npm run dev` → Next.js 개발 서버 실행 (포트 3000)
2. `http://localhost:3000` 메인 페이지 접속 확인
3. `npm run build` → TypeScript 컴파일 오류 없음 확인
