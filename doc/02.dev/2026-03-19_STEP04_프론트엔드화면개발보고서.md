### 화면 01. 로그인 페이지 `/login`

**파일**: `src/app/(auth)/login/page.tsx`, `src/components/auth/LoginForm.tsx`

#### 기능
| 기능 | 설명 |
|------|------|
| 이메일/비밀번호 입력 | react-hook-form + zod 유효성 검사 |
| 로그인 요청 | `POST /api/v1/auth/login` |
| 토큰 저장 | accessToken → sessionStorage, refreshToken → localStorage |
| Zustand 상태 갱신 | `setUser()`, `setAccessToken()` |
| 성공 시 이동 | isAdmin이면 `/admin`, 일반이면 `/` |
| 실패 시 오류 표시 | 서버 메시지 또는 기본 오류 메시지 |

#### UI 구성
- 이메일 Input (type=email, @email.com 포맷 검증)
- 비밀번호 Input (type=password, 최소 1자)
- 서버 오류 표시 박스 (빨간 테두리)
- 로그인 버튼 (로딩 스피너 포함)
- 하단 링크: 회원가입 | 비밀번호 찾기

#### 기술 결정 사항
- Zustand + sessionStorage 이중 관리: 컴포넌트 상태는 Zustand, 브라우저 새로고침 후 복원은 sessionStorage의 값을 axios 인터셉터가 직접 참조
- next-auth SessionProvider는 Google OAuth 확장을 위해 유지하되, 현재 로그인은 커스텀 JWT 방식 사용

## 기술 스택 및 설계 결정

### 상태 관리
| 유형 | 도구 | 용도 |
|------|------|------|
| 서버 상태 | TanStack React Query | API 데이터 캐싱, 무효화, 로딩 상태 |
| 전역 클라이언트 상태 | Zustand | 인증 정보(user, accessToken), 장바구니 수량 |
| 로컬 UI 상태 | useState | 선택 옵션, 모달 열림 여부, 페이지 등 |

### 폼 처리
- react-hook-form + zod: 타입 안전한 스키마 기반 유효성 검사
- `zodResolver(schema)` 연동으로 에러 메시지 자동 바인딩

### API 호출
- axios 인터셉터: accessToken 자동 주입, 401 시 refreshToken으로 재시도
- TanStack Query: `queryKey` 배열로 캐시 무효화 범위 제어

### 인증 흐름
```
로그인 → accessToken(sessionStorage) + refreshToken(localStorage)
API 요청 → axios 인터셉터에서 Bearer 토큰 주입
401 발생 → refreshToken으로 재발급 → 원래 요청 재시도
로그아웃 → 서버 블랙리스트 + 클라이언트 토큰 삭제 + Zustand 초기화
```
