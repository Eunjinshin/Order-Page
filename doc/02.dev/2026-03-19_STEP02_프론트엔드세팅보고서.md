## 피드백 내용

PRD v1.0 기반 전체 라우팅 구조와 핵심 기반 파일을 Next.js 14 App Router 방식으로 구성하였다.
백엔드 API 연동을 위한 axios 클라이언트(JWT 자동 갱신 포함), TypeScript 타입 정의, 상태관리(zustand), 공통 레이아웃을 세팅하였다.

---

## 수정 사항 (생성된 파일 목록)

### 프로젝트 설정
| 파일 | 설명 |
|------|------|
| `frontend/package.json` | Next.js 14.2.29, React 18, 전체 의존성 |
| `frontend/tsconfig.json` | TypeScript strict 모드, `@/*` 경로 별칭 |
| `frontend/next.config.ts` | S3 이미지 도메인 허용, API rewrites |
| `frontend/tailwind.config.ts` | 커스텀 컬러(primary/brand), Pretendard 폰트 |
| `frontend/postcss.config.js` | Tailwind + autoprefixer |
| `frontend/.eslintrc.json` | next/core-web-vitals 규칙 |
| `frontend/.env.local.example` | 환경변수 템플릿 (API URL, NextAuth, Google OAuth) |

## JWT 토큰 관리 전략

| 토큰 | 저장소 | 만료 |
|------|--------|------|
| Access Token | sessionStorage | 1시간 |
| Refresh Token | localStorage | 30일 |

401 응답 시 axios 인터셉터가 자동으로 Refresh Token으로 재발급 → 재요청 처리.

## 미결 해결 과제

| 우선순위 | 항목 | 설명 |
|:--------:|------|------|
| 높음 | npm install 실행 필요 | `cd frontend && npm install` 실행 후 개발 시작 가능 |
| 높음 | .env.local 생성 필요 | `.env.local.example` 복사 후 실제 값 입력 |
| 높음 | NextAuth 설정 미완성 | `src/app/api/auth/[...nextauth]/route.ts` 구현 필요 (인증 기능 개발 단계) |
| 중간 | Google OAuth Client ID/Secret | Google Cloud Console에서 발급 필요 |
| 중간 | 나머지 페이지 컴포넌트 구현 | 각 페이지 TODO 주석 기준으로 기능별 개발 예정 |
| 낮음 | 반응형/접근성 검증 | WCAG 2.1 AA 레벨 검증은 기능 구현 후 진행 |

## 로컬 실행 방법 (세팅 완료 후)

```bash
# 1. 의존성 설치
cd frontend
npm install

# 2. 환경변수 설정
cp .env.local.example .env.local
# .env.local 파일을 열어 실제 값 입력

# 3. 개발 서버 실행
npm run dev

# 4. 확인
# 메인 페이지: http://localhost:3000
# 로그인: http://localhost:3000/login
```
