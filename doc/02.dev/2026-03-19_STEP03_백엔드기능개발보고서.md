## 주요 기술 결정 사항

### JWT 인증
- Access Token 1시간 / Refresh Token 30일
- 로그아웃 시 Access Token Redis 블랙리스트 처리
- 로그인 5회 실패 시 Redis 카운터로 30분 잠금

### 주문 설계
- `ORDER_ITEM.unit_price` = 주문 시점 가격 스냅샷 (가격 변경에 무관)
- 쿠폰: RATE(비율 할인) / FIXED(정액 할인), `is_used` 플래그로 중복 사용 방지
- 재고 차감: `@Transactional` DB 레벨 (Redis 분산 락은 TODO)

### 결제 (Mock PG)
- `confirmPayment()`: UUID로 임시 `pgTransactionId` 생성 (실제 Tosspayments 연동 미구현)
- 환불: `processRefund()` TODO — PG사 API 연동 필요

### 보안
- Admin 엔드포인트: `hasRole("ADMIN")` → JWT `isAdmin` 클레임 기반
- 공개 엔드포인트: GET /products/**, GET /categories/**, POST /auth/**
