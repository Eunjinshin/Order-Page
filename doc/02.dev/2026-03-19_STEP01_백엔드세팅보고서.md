## 피드백 내용

ERD v1.0(23개 테이블) 및 PRD v1.0을 기반으로 Spring Boot 3.x 백엔드 프로젝트의 기반 구조를 생성하였다.
코드 구현이 전혀 없는 상태에서 시작하여, 프로젝트 초기화에 필요한 모든 설정 파일과 핵심 소스 파일을 작성하였다.

---

## 수정 사항 (생성된 파일 목록)

### 인프라 / 환경 설정
| 파일 | 설명 |
|------|------|
| `docker-compose.yml` | MySQL 8.0 + Redis 7.x 로컬 개발 환경 구성 |
| `backend/build.gradle` | Spring Boot 3.3.5, Java 17, 전체 의존성 정의 |
| `backend/settings.gradle` | 루트 프로젝트명: odersite |
| `backend/gradle/wrapper/gradle-wrapper.properties` | Gradle 8.10.2 래퍼 설정 |

### 애플리케이션 설정
| 파일 | 설명 |
|------|------|
| `backend/src/main/resources/application.yml` | DB/Redis/JWT/Mail/Swagger 공통 설정 (환경변수 기반) |
| `backend/src/main/resources/application-local.yml` | 로컬 개발용 설정 (SQL 로그 활성화) |

### DB 마이그레이션
| 파일 | 설명 |
|------|------|
| `backend/src/main/resources/db/migration/V1__init_schema.sql` | ERD v1.0 기준 23개 테이블 전체 DDL + 인덱스 정의 |

**테이블 생성 순서 (FK 의존성 순)**:
1. MEMBER_USER → AUTH_LOGIN → MEMBER_PROFILE → MEMBER_ADDRESS
2. SHOP → CATEGORY → PRODUCT → PRODUCT_OPTION → PRODUCT_IMAGE → STOCK_HISTORY
3. COUPON → USER_COUPON
4. CART → ORDERS → ORDER_ITEM → RETURN_REQUEST
5. PAYMENT → REFUND → DELIVERY
6. REVIEW → REVIEW_IMAGE → REVIEW_HELPFUL → REVIEW_REPLY

### Java 소스 파일
| 파일 | 설명 |
|------|------|
| `OdersiteApplication.java` | 메인 진입점 (@SpringBootApplication, @EnableJpaAuditing) |
| `global/entity/BaseTimeEntity.java` | JPA Auditing 공통 엔티티 (createdAt, updatedAt) |
| `global/response/ApiResponse.java` | 공통 API 응답 포맷 (success, message, data) |
| `global/exception/ErrorCode.java` | 에러 코드 열거형 (HTTP 상태 + 메시지) |
| `global/exception/BusinessException.java` | 비즈니스 예외 기반 클래스 |
| `global/exception/GlobalExceptionHandler.java` | 전역 예외 처리기 (@RestControllerAdvice) |
| `domain/member/entity/MemberUser.java` | 회원 엔티티 |
| `domain/member/repository/MemberUserRepository.java` | 회원 JPA 레포지토리 |
| `domain/auth/entity/AuthLogin.java` | 인증 엔티티 (이메일/소셜, Refresh Token, 소프트 딜리트) |
| `domain/auth/repository/AuthLoginRepository.java` | 인증 JPA 레포지토리 |
| `config/SecurityConfig.java` | Spring Security (STATELESS, CSRF 비활성화) |
| `config/RedisConfig.java` | Redis Template 설정 |
| `config/SwaggerConfig.java` | SpringDoc OpenAPI 설정 (JWT Bearer Auth) |

## 미결 해결 과제

| 우선순위 | 항목 | 설명 |
|:--------:|------|------|
| 높음 | Gradle Wrapper 바이너리 | `gradle wrapper --gradle-version=8.10.2` 명령 실행 필요 (gradlew 바이너리는 직접 생성 불가) |
| 높음 | 도메인별 Entity 미완성 | 현재 MemberUser, AuthLogin만 구현 — 나머지 21개 엔티티는 기능 개발 시 순차 추가 |
| 중간 | JWT 유틸 구현 미완성 | JwtUtil 클래스 작성 필요 (인증 기능 개발 단계에서 구현) |
| 중간 | PG사 선택 미확정 | Tosspayments vs KG Inicis — 결제 기능 구현 단계에서 확인 필요 |
| 낮음 | AWS 인프라 미구성 | 로컬 개발 → 개발 서버 → 프로덕션 순서로 진행 예정 |
| 낮음 | 환경변수 관리 | `.env` 파일 또는 AWS Secrets Manager 활용 방식 확정 필요 |

## 로컬 실행 방법 (세팅 완료 후)

```bash
# 1. Docker로 MySQL + Redis 실행
docker-compose up -d

# 2. Gradle Wrapper 생성 (최초 1회)
cd backend
gradle wrapper --gradle-version=8.10.2

# 3. 서버 실행
./gradlew bootRun --args='--spring.profiles.active=local'

# 4. 확인
# Swagger UI: http://localhost:8080/swagger-ui.html
# API Docs: http://localhost:8080/v3/api-docs
```
