## 1. 시스템 개요

Odersite는 상품 탐색 → 장바구니 → 주문·결제 → 배송 → 리뷰까지 전체 커머스 플로우를 지원하는 웹 플랫폼이다. 본 ERD는 해당 시스템의 논리 데이터 모델을 정의한다.

### 1.1 설계 원칙

- **3NF 기반 정규화**: 이행 함수 종속 제거, 도메인별 테이블 분리
- **소프트 딜리트**: 회원 탈퇴 시 물리 삭제 대신 `deleted_at` 타임스탬프 사용
- **가격 스냅샷**: `ORDER_ITEM.unit_price`에 주문 시점 단가 저장으로 가격 변경 영향 차단
- **계층 카테고리**: `CATEGORY.parent_id` 자기 참조로 대/중/소 3단계 표현
- **옵션별 재고**: `PRODUCT_OPTION`에서 색상·사이즈 단위 재고 관리 및 변동 이력 기록

## 3. 테이블 상세 정의

> **표기 규칙**
> - `PK` : 기본키 / `FK` : 외래키 / `PK·FK` : 복합키
> - `NN` : NOT NULL / `UQ` : UNIQUE

| 컬럼명 | 타입 | 키 | NN | 설명 |
|--------|------|:--:|:--:|------|
| `user_id` | INT | FK | Y | 회원 식별자 → MEMBER_USER |
| `email` | VARCHAR | | Y | 로그인 이메일 |
| `password_hash` | VARCHAR | | | bcrypt 해시 비밀번호 (소셜 로그인 시 NULL) |
| `refresh_token` | VARCHAR | | Y | Refresh Token (유효기간 30일) |
| `token_expires_at` | DATETIME | | | 토큰 만료 일시 |
| `login_type` | ENUM | | | EMAIL \| GOOGLE |
| `created_at` | DATETIME | | Y | 가입 일시 |
| `deleted_at` | DATETIME | | | 탈퇴 일시 (소프트 딜리트) |

> **설계 노트**: Access Token은 stateless 방식으로 서버에 저장하지 않으며 `refresh_token`만 DB에 보관한다.

## 5. 인덱스 설계

### 5.1 필수 인덱스 (성능 필수)

```sql
-- 주문 목록 조회 (회원별, 최신순)
CREATE INDEX idx_orders_user_ordered
  ON ORDERS (user_id, ordered_at DESC);

-- 결제·환불 조인 경로
CREATE INDEX idx_payment_order_id    ON PAYMENT (order_id);
CREATE INDEX idx_refund_payment_id   ON REFUND (payment_id);

-- 재고 이력 조회
CREATE INDEX idx_stock_option_created
  ON STOCK_HISTORY (option_id, created_at DESC);

-- 장바구니
CREATE INDEX idx_cart_user_id        ON CART (user_id);

-- 리뷰 상품별 조회
CREATE INDEX idx_review_product
  ON REVIEW (product_id, is_blind, created_at DESC);

-- 배송 조회
CREATE INDEX idx_delivery_order_id   ON DELIVERY (order_id);
CREATE INDEX idx_delivery_tracking   ON DELIVERY (tracking_number);
```

### 5.2 중복 방지 유니크 인덱스

```sql
-- 장바구니: 동일 회원+옵션 중복 방지
CREATE UNIQUE INDEX uq_cart_user_option
  ON CART (user_id, option_id);

-- 도움돼요: 동일 회원이 동일 리뷰에 중복 표시 방지
CREATE UNIQUE INDEX uq_helpful_review_user
  ON REVIEW_HELPFUL (review_id, user_id);

-- 쿠폰: 동일 회원에게 동일 쿠폰 중복 발급 방지
CREATE UNIQUE INDEX uq_user_coupon
  ON USER_COUPON (user_id, coupon_id);
```

### 5.3 인덱스 우선순위 요약

| 우선순위 | 대상 | 이유 |
|:--------:|------|------|
| P1 | ORDERS (user_id, ordered_at) | 주문 목록 가장 빈번한 쿼리 |
| P1 | STOCK_HISTORY (option_id, created_at) | 무한 성장 테이블 |
| P1 | PAYMENT (order_id) | 주문-결제 조인 필수 경로 |
| P2 | REVIEW (product_id, is_blind, created_at) | 상품 상세 리뷰 조회 |
| P2 | CART (user_id) | 매 페이지 진입 시 조회 |
| P3 | DELIVERY (tracking_number) | 배송 조회 API |
